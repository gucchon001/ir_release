解析するPythonコードの内容を日本語で記述し、プログラムの機能要件仕様書を作成してください。

# 機能要件仕様書の構成:

- システムの概要と目的
- プログラムが持つ主要機能
- 非機能要件（パフォーマンス要件、セキュリティ要件、可用性など）
- 技術的な要件と依存関係

コード解析には注意深い確認を行い、それに基づいて以下の各項目を埋めてください。

# 機能要件仕様書の各項目

### 1. システム概要
プログラムが提供する目的やシステム全体の概要を記述してください。

### 2. 主要機能要件
プログラム内で提供される機能を、個別に説明してください。各関数やクラスが行うタスクの目的について詳細に、かつわかりやすく記述してください。

### 3. 非機能要件
パフォーマンスの要求など非機能的な制約・要件（例えば実行時間やメモリ要件等）を記述してください。また、セキュリティ要件や、依存しているライブラリについても言及します。

### 4. 技術要件
プログラムを適切に実行するために必要な技術的な条件について説明してください（例：言語のバージョン、外部ライブラリ、OS要件など）。

# Output Format

出力は日本語で明確に書かれたテキストとして以下の形式に従い、詳細に記述してください。

1. システム概要
    - プログラムの全体的な目的と対象のユーザー。
2. 主要機能要件
    - 提供される各機能の説明。
3. 非機能要件
    - パフォーマンス、セキュリティなどの項目を具体化。
4. 技術要件
    - 開発環境、システム環境、必要なライブラリ等の説明。

# Notes

- 解析を行う際には、コード内の関数やクラスがどのように相互作用しているかも考慮してください。
- 同じ機能を持つ複数の部品がある場合、それらは統括して説明して構いません。
- 結果はドキュメント形式で読みやすくなるように、過度に技術的でなく、日本語でわかりやすく記述してください。
